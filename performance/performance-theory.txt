================= performance.now() Theory ============================
================= DeepSeek ============================

performance.now() is a high-resolution timestamp method that returns the current time in milliseconds (with microsecond precision) since the page started loading.

In your code, it's being used as a performance measurement tool to log how long the search operation takes:

javascript
function searchData(query){
    console.log("search Performance", performance.now());  // Logs the timestamp
    
    // ... rest of the search logic
}
What it's measuring:
When you type in the search input, this logs the exact timestamp when searchData() is called. If you wanted to measure the duration of the search operation, you'd typically do:

javascript
function searchData(query){
    const startTime = performance.now();  // Start time
    
    searchListElm.innerHTML = "";
    const filteredData = data
        .filter((value)=> value.includes(query))
        .slice(0, 10);
    
    filteredData.forEach((value, index)=>{
        const li = document.createElement("li");
        li.textContent = value;
        searchListElm.append(li);
    });
    
    const endTime = performance.now();  // End time
    console.log("Search took:", endTime - startTime, "milliseconds");
}
Key differences from Date.now():
performance.now(): Higher precision (microseconds), monotonic (always increases), ideal for measuring performance

Date.now(): Millisecond precision, affected by system clock changes, better for absolute timestamps

Current usage:
Right now, your code is just logging the timestamp, not measuring the actual search duration. If you want to see how fast your search is performing, use the start/end approach shown above!

explain for beginners
I'll explain performance.now() in simple terms for beginners! 🎯

What is performance.now()?
Think of it like a stopwatch that starts when your webpage loads. It tells you exactly how many milliseconds have passed since the page opened.

javascript
console.log(performance.now()); // Something like: 1234.567 milliseconds
Real-World Analogy
Imagine you're timing how fast you can find a book in a library:

javascript
// START stopwatch when you begin searching
const startTime = performance.now();

// Do your search (find the book)
const filteredData = data.filter(item => item.includes("item5"));

// STOP stopwatch when you're done
const endTime = performance.now();

// See how long it took
console.log("Search took:", endTime - startTime, "milliseconds");
// Output: Search took: 0.234 milliseconds
In Your Code
Your code is currently doing this:

javascript
function searchData(query){
    console.log("search Performance", performance.now());
    // This just prints: "search Performance 1234.567"
    // It's like saying "I started searching at 1.23 seconds after page load"
}
But it's NOT measuring how long the search takes. To measure the duration, you need to subtract the start time from the end time:

javascript
function searchData(query){
    const start = performance.now();  // 👈 Start timing
    
    // Your search logic here...
    searchListElm.innerHTML = "";
    const filteredData = data
        .filter((value)=> value.includes(query))
        .slice(0, 10);
    
    filteredData.forEach((value)=>{
        const li = document.createElement("li");
        li.textContent = value;
        searchListElm.append(li);
    });
    
    const end = performance.now();    // 👈 End timing
    console.log("Search took:", end - start, "milliseconds");
    // Output: Search took: 0.05 milliseconds (very fast!)
}
Why is this useful?
Find slow code - If a search takes 100ms, users might notice lag

Compare methods - See if filtering 100 items is faster than 1000 items

Optimize performance - Make your app feel snappy!

Simple Example You Can Try
javascript
// Measure how long a loop takes
const start = performance.now();

for(let i = 0; i < 1000000; i++) {
    // Do something 1 million times
    Math.sqrt(i);
}

const end = performance.now();
console.log("Loop took:", end - start, "milliseconds");
// Output: Loop took: 15.2 milliseconds
Key Points for Beginners
performance.now() = "How many milliseconds since page loaded?"

Use start and end to measure time differences

Great for finding out what parts of your code are slow

Much more precise than using Date.now() for timing

In your search app, this helps you understand if your search is fast enough when users type! 🚀
